# About This Example

This example shows `repository-surgeon`'s output after acting on the
audit in [`../sample-audit/`](../sample-audit/) for the same fictional
TaskFlow API project. It demonstrates `REFACTOR_SUMMARY.md` — the
roll-up document, written after every individual change has already
been logged in detail in `REFACTOR_LOG.md` (omitted here for brevity;
see `skills/repository-surgeon/templates/REFACTOR_LOG_TEMPLATE.md` for
its exact per-change structure).
