# About This Example

This example shows `repository-guardian`'s output after independently
re-verifying the audit, refactor, and documentation work shown in the
other three example folders for the same fictional TaskFlow API
project. Note how `FINAL_REVIEW.md` references specific entries from
the earlier stages by name rather than restating them wholesale, and
how `QUALITY_SCORECARD.md` re-derives its own numbers rather than
copying `REPOSITORY_SCORECARD.md`'s forward unchanged.
